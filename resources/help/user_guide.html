Getting Started

Overview

WireTrace is a professional serial data monitor and logger designed for embedded engineers. It captures, classifies, and exports data from serial devices in real time.

Key capabilities:

  • Monitor multiple serial ports simultaneously in separate tabs
  • Sustained throughput of 10,000+ lines per second
  • Automatic severity-based message classification (7 tags)
  • Real-time search and line filtering
  • Export to .txt and .csv with structured column detection
  • Light and Dark themes
  • Cross-platform — Windows, macOS, and Linux

Your First Connection

  1. Connect your serial device (USB, FTDI, etc.) to the computer.
  2. Launch WireTrace. You'll see the welcome screen with a single button: Connect a Device.
  3. Click it. A new tab opens with the connection panel.
  4. Select your port from the Port dropdown. Click Refresh if your device doesn't appear.
  5. Select the baud rate (default: 115200). You can type a custom rate directly.
  6. Click Connect.

Data from the device appears immediately in the console.

Tip The port dropdown shows the device description and manufacturer (e.g., "COM3 — Arduino Uno (Arduino LLC)") to help you identify the correct port.

Your First Log

  1. Once connected, click Log On in the toolbar.
  2. The New Log dialog opens. Enter a session name (e.g., "MotorTest").
  3. Choose a save directory and export format (.txt, .csv, or both).
  4. Optionally add a description.
  5. Click Start Logging. All incoming data is now written to disk.
  6. Click Log Off to stop and close the log files.
Note Logging captures every line from the serial port, regardless of any active filters on the console display. Your log files are always complete.

Interface

Multi-Device Tabs

WireTrace supports multiple simultaneous serial connections. Each device runs in its own tab with completely isolated data, threads, and log files.

  • Click the + button (or Ctrl+T) to create a new tab.
  • Each tab has its own connection panel, console, log engine, and status bar.
  • Closing a tab with active logging prompts for confirmation, then performs an ordered shutdown — data is flushed before the port is released.
  • Right-click a tab header to Rename or Close it.
  • When all tabs are closed, the welcome screen reappears.

Console Display

The console is a high-performance virtual viewport. It renders only the visible lines, keeping memory usage constant regardless of total data volume. The full log is always on disk.

  • Auto-scroll — Enabled by default. New lines scroll into view as they arrive. Scrolling up manually pauses auto-scroll; scrolling back to the bottom re-enables it.
  • Color mode — Messages are color-coded by severity (see Message Classification).
  • Clear — Clears the console display. Does not affect log files.

Command Bar

The command input at the bottom of each tab sends data to the connected device. Type a command and press Enter or click Send.

  • Command history is accessible with ↑ and ↓ arrow keys.
  • Sent commands appear in the console tagged as COMMAND.

Export

Use File → Export… (Ctrl+E) to save the current console contents to a file. This is useful when you forgot to start logging and want to capture what's on screen.

  1. Enter a file name.
  2. Choose a directory.
  3. Select the format: .txt, .csv, or both.

Quick Save

Press Ctrl+S to instantly save the console contents to a .txt file. No dialog — the file is auto-named and saved to your default log directory. A notification confirms the save.

CSV Modes

WireTrace detects the structure of your serial data and applies one of two CSV modes:

ModeDetectionOutput
Auto-detect Data contains key:value, key=value pairs, or JSON objects. Keys become column headers. Values are placed in the correct columns.
Raw Unstructured data (no consistent pattern detected). Two columns: Timestamp and Data.

Auto-detect examines the first 50 lines of data. Example structured output:

Timestamp,Temperature,Humidity,Pressure
2026-02-25 14:30:01.123,25.3,60.1,1013.2
2026-02-25 14:30:01.456,25.4,60.0,1013.1

File Naming

Log and export files follow a consistent naming convention:

{Name}_{Port}_{YYYY-MM-DD}_{HH-MM-SS}.{ext}

Examples:

MotorTest_COM3_2026-02-25_14-30-00.txt
SensorCalibration_COM7_2026-02-25_09-15-30.csv

Search & Filter

Search and Filter controls sit on the same row at the bottom of the console area. The filter is always visible; the search appears to its right when activated.

Search

Press Ctrl+F to activate the search field. Type a term to highlight all matching occurrences in the console.

  • F3 — Jump to next match.
  • Shift+F3 — Jump to previous match.
  • Escape — Close search and clear highlights.
  • The match counter shows your position (e.g., "3/47").

Filter

The filter field is always visible. Type a term to hide all non-matching lines from the console in real time. Press Ctrl+L to focus the filter field quickly.

Message Classification

WireTrace automatically classifies incoming serial messages into 7 severity levels based on keyword detection. Each tag is displayed with a distinct color.

TagTrigger KeywordsExample
CRITICAL fatal, critical, panic, crash CRITICAL: Watchdog reset — system halted
ERROR error, fail, exception, fault ERROR: I2C timeout on sensor 0x48
WARNING warn, warning, caution WARNING: Battery voltage below threshold
INFO info, connected, ready, success, ok, initialized, complete INFO: WiFi connected to AP
DEBUG debug, trace, verbose DEBUG: Buffer utilization at 23%
COMMAND Lines sent from the command bar >>> AT+RST
DATA Everything else Temperature: 25.3°C

Classification uses priority ordering — the most severe matching tag wins.

Timestamps

WireTrace supports two timestamp display modes. Toggle by clicking the timestamp indicator in the status bar, or via View → Timestamp Format.

ModeIndicatorConsole Display
Timestamp 🕐 Timestamp [2026-02-25 14:30:01.123]
Elapsed Δt Elapsed [+Δ 0.003s] (time since previous line)

This setting affects the console display only. Log files always use full timestamps.

Text & HEX Mode

Switch between Text and HEX display modes from View → Display Mode. Text mode decodes incoming bytes as UTF-8. HEX mode displays raw byte values, which is useful for debugging binary protocols.

Preferences

Open preferences with Ctrl+, or File → Preferences. Settings include default baud rate, default log directory, export format, theme, font size, display options, and performance tuning. All preferences persist across sessions.

Auto-Update

WireTrace checks for updates automatically on startup. When a new version is available, a dialog presents three options:

  • Update Now — Downloads the installer with a live progress bar, verifies its integrity (SHA-256), then prompts to install. Clicking Install & Restart launches the installer and closes WireTrace.
  • Remind Later — Snoozes the notification for 24 hours.
  • Skip This Version — Suppresses the notification for this specific version.

You can also check manually via Help → Check for Updates.

Troubleshooting

Port not listed

Click Refresh to re-scan. Ensure the device is plugged in and drivers are installed. On Linux, your user may need to be in the dialout group: sudo usermod -aG dialout $USER (then log out and back in).

Garbled or unreadable data

This almost always indicates a baud rate mismatch. Confirm the correct rate in your device's documentation. Try common rates: 9600, 115200.

Data appears to lag or stutter

At very high data rates (>5,000 lines/sec), ensure no other application is competing for the same serial port. Close any other serial monitors. WireTrace batches GUI updates at 50ms intervals to maintain responsiveness.

Log file is empty

Log files are buffered for performance. Data is flushed every second and on Log Off. If the application is terminated unexpectedly, the last second of data may be lost. Always use Log Off or close the tab properly.

CSV has only two columns

This means your serial data was not detected as structured. Auto-detect requires consistent key:value, key=value, or JSON formatting. If your data is unstructured, the Raw mode (Timestamp, Data) is the correct output.

Resetting preferences

If WireTrace behaves unexpectedly, you can reset all settings by deleting the preferences file (see File Locations) and restarting the application.
